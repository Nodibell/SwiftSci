## 2. Модуль SwiftLLM (Генеративний ШІ)

Модуль `SwiftLLM` дозволяє запускати локальні мовні моделі на Apple Silicon з використанням MLX.

### Ключові компоненти:
- **`LLMModel` (Protocol):**
  - Визначає інтерфейс для генерації токенів:
    ```swift
    public protocol LLMModel: Sendable {
        func generate(prompt: String, options: LLMOptions) async throws -> AsyncStream<String>
    }
    ```
- **`TransformerDecoder` (Class / Actor):**
  - Легковажна реалізація архітектури Transformer (Decoder-only) з використанням `MLXNN.Module`.
  - Підтримка шарів `MultiHeadAttention`, `RMSNorm`, `Linear` та проекції токенів.
- **`LLMOptions` (Struct):**
  - Конфігурація генерації: `temperature` (температура), `topP` (nucleus sampling), `topK`, `maxTokens`.
- **Sampler:**
  - Реалізація確率ного вибору наступного токена на основі розподілу ймовірностей (Logits).

## 3. Модуль SwiftExplain (Інтерпретація моделей)

Модуль `SwiftExplain` реалізує алгоритми пояснення рішень моделей "чорної скриньки".

### Ключові компоненти:
- **`KernelSHAP` (Actor):**
  - Реалізація методу Kernel SHAP (Shapley Additive exPlanations).
  - Алгоритм:
    1. Генерація коаліцій ознак (бінарних масок).
    2. Інтерполяція пропущених ознак за допомогою фонового датасету (background dataset).
    3. Оцінка прогнозів моделі для кожної коаліції.
    4. Розрахунок ваг Шаплі для кожної коаліції:
       $$W(S) = \frac{M - 1}{\binom{M}{|S|} |S| (M - |S|)}$$
    5. Вирішення зваженої лінійної регресії для визначення SHAP-значень (коефіцієнтів).
- **Паралельне обчислення:**
  - Оцінка прогнозів моделі виконується конкурентно через `TaskGroup` на CPU для оптимізації швидкодії.

## 4. Модуль SwiftPrivacy (Конфіденційність та PNNS)

Модуль `SwiftPrivacy` забезпечує виконання обчислень над зашифрованими даними без їх розшифрування (Homomorphic Encryption).

### Ключові компоненти:
- **Приватний пошук найближчого сусіда (Private Nearest Neighbor Search - PNNS):**
  - На основі схеми BFV (Brakerski-Fan-Vercauteren) з `swift-homomorphic-encryption`.
- **`PNNSClient` (Struct):**
  - Генерує секретний та публічний ключі.
  - Шифрує вектор запиту (query vector) клієнта: $\text{Enc}(q)$.
  - Розшифровує отримані результати відстаней від сервера.
- **`PNNSServer` (Struct):**
  - Зберігає базу даних векторів у відкритому вигляді.
  - Обчислює гомоморфний скалярний добуток (Homomorphic Dot Product) між зашифрованим запитом клієнта та базою даних сервера:
    $$\text{Enc}(\langle q, x_i \rangle) = \sum_j \text{Enc}(q_j) \cdot x_{i,j}$$
  - Повертає масив зашифрованих відстаней клієнту.

## 5. Оновлення Package.swift та залежностей

У файлі `Package.swift` будуть проведені наступні зміни:

1. Додавання нової залежності:
   ```swift
   .package(
       url: "https://github.com/apple/swift-homomorphic-encryption.git",
       from: "1.0.0"
   )
   ```
2. Визначення нових бібліотек у `products`:
   - `.library(name: "SwiftLLM", targets: ["SwiftLLM"])`
   - `.library(name: "SwiftExplain", targets: ["SwiftExplain"])`
   - `.library(name: "SwiftPrivacy", targets: ["SwiftPrivacy"])`
3. Визначення нових таргетів та тест-таргетів:
   - `SwiftLLM` (залежності: `SwiftNLP`, `mlx-swift`)
   - `SwiftExplain` (залежності: `SwiftML`, `SwiftStats`)
   - `SwiftPrivacy` (залежність: `swift-homomorphic-encryption`)

## 6. План верифікації та тестування

### 6.1 Автоматичні тести
- **SwiftLLMTests:**
  - Тест ініціалізації шарів уваги та декодера.
  - Тест генераційного циклу зі збіжністю токенів.
- **SwiftExplainTests:**
  - Розрахунок SHAP-значень для простої лінійної моделі та порівняння з реальними вагами (SHAP значення мають точно відповідати коефіцієнтам лінійної регресії).
  - Тест суми SHAP-значень (сума SHAP-значень ознак плюс базове значення має дорівнювати прогнозу моделі для даного входу).
- **SwiftPrivacyTests:**
  - Тест шифрування, гомоморфного множення та дешифрування.
  - Валідація знаходження правильного індексу найближчого сусіда на зашифрованому запиті.

### 6.2 Інтеграційні тести
- Запуск повного сьюту тестів (`swift test`) для перевірки Strict Concurrency та відсутності конфліктів залежностей.
- Додавання бенчмарків швидкодії нових модулів до `SwiftAnalyticsBenchmarks`.
